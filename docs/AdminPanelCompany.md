# Admin Panel — Компани удирдлага (AdminPanelCompany) — Phase 1 spec

Зорилго: **Super Admin** болон (шаардлагатай бол) бусад staff-ийн **админ панел** дээрх компани холбоотой дэлгэц, урсгал, API хамаарлыг тодорхойлох. Backend нь `docs/addCompany.md`-тай нийцнэ.

---

## 1. Хэрэглэгчийн дүр

| Дүр | Хандалт |
|-----|---------|
| **Super Admin** | Компанийн бүртгэл бүхэлд нь: үүсгэх, засах, статус, жагсаалт, нэг компанийн дэлгэрэнгүй. |
| **Admin / CEO / Manager** | Тодорхой бодлого: жишээ нь зөвхөн **унших** эсвэл **хориглох** — бүтээгдэхүүний баг шийднэ; spec-д "тохируулгаар" гэж тэмдэглэнэ. |
| **Company operator** | Өөрийн компанийн **бараа** хэсэг (тусад баримт бичигтай нийцэх); энэхүү баримт бичигт голчлон **Super Admin-ийн компани удирдлага**. |

---

## 2. Навигаци / цэс

- Гол цэсэнд **"Компаниуд"** (`/admin/companies` эсвэл төслийн routing-тай нийцнэ).
- Дэд цэс эсвэл таб: **Бүгд** | **Идэвхтэй** | **Хүлээгдэж буй** | **Түдгэлзсэн** (статусын filter-тэй нэг дэлгэц ч болно).

---

## 3. Дэлгэцүүд (wireframe-level)

### 3.1. Компанийн жагсаалт

| Элемент | Тайлбар |
|---------|---------|
| Хүснэгт | Нэр, slug, имэйл, утас, статус, барааны тоо (тоо Phase 2 API), үүсгэсэн огноо. |
| Хайлт | Нэр, имэйл, slug. |
| Pagination | Хуудаслалт стандарт. |
| Үйлдэл | "Дэлгэрэнгүй", "Засах" (Super Admin). |

### 3.2. Шинэ компани нэмэх (`add company`)

Формын талбарууд (`addCompany.md` дахь `companies` schema-тай нийцнэ):

- Албан нэр (`name`) — заавал.
- `slug` — заавал, unique; автоматаар латинжуулах санал (frontend validation).
- `legalName`, `email`, `phone`, `logo` (upload — одоогийн Cloudinary урсгалтай нийцүүлэх боломжтой).
- Хадгалах үед: `POST` амжилттай бол жагсаалт руу буцаах эсвэл дэлгэрэнгүй рүү шилжих.

### 3.3. Компанийн дэлгэрэнгүй

| Хэсэг | Агуулга |
|-------|---------|
| Ерөнхий мэдээлэл | Бүх талбар унших. |
| Статус | Super Admin: `active` / `suspended` / `pending` сонголт (баталгаажуулах модал). |
| Холбоотой бараа | Тухайн `companyId`-тай барааны жагсаалт (Phase 2: `GET .../companies/:id/products`). |
| Ажилтнууд | Хэрэв Company operator-ийг `Admin` + `companyId`-аар явуулбал: энэ компанид харьяалагдсан админуудын жагсаалт (Phase 2). |

### 3.4. Компанийн хүрээнд бараа (staff эсвэл company operator)

- Одоогийн **бараа нэмэх / засах** формыг ашиглах бол **dropdown эсвэл нуугдмал** `companyId`: Super Admin-д бүх компани сонгох; company operator-д **зөвхөн өөрийн компани** (disabled/hidden талбар).

---

## 4. Алдаа, аюулгүй байдал

- **403**: Super Admin биш хүн компани үүсгэхэд.
- **409**: `slug` эсвэл `email` давхардсан.
- Токен дуусах: одоогийн админ login урсгалыг дагана.

---

## 5. Backend API хамаарол (шалгах жагсаалт)

| Method | Path (санал) | Middleware |
|--------|----------------|------------|
| GET | `/api/companies` | `requireSuperAdmin` |
| POST | `/api/companies` | `requireSuperAdmin` |
| GET | `/api/companies/:id` | `requireSuperAdmin` (эсвэл staff read) |
| PATCH | `/api/companies/:id` | `requireSuperAdmin` |

Барааны endpoint-уудыг `addCompany.md`-ийн "Company scope" хэсэгтэй нэгтгэнэ.

---

## 6. Phase 1 vs Phase 2

| Phase 1 (одоо) | Phase 2 |
|-----------------|---------|
| Дэлгэцийн бүтэц, талбар, эрхийн хүснэгт | API холболт, жинхэнэ өгөгдөл, барааны тоо, upload |
| Super Admin-only компани CRUD-ийн тохиролцоо | Company operator нэвтрэлт, нууц сэргээлт |

---

## 7. Acceptance criteria (нарийвчилсан шалгах)

1. Super Admin нэвтэрсэн үед **Компаниуд** цэс харагдана; бусад role-д нуугдах эсвэл disabled — тохиргоогоор.
2. Жагсаалт хуудас хоосон үед хоосон төлөв + "Компани нэмэх" CTA.
3. Форм дээр заавал талбарууд тодорхой, алдааны мессеж backend-тай нийцтэй (`slug` давхардал г.м.).
4. Компани статус өөрчлөхөд UI дахь төлөв шинэчлэгдэнэ (optimistic эсвэл refetch — Phase 2).
